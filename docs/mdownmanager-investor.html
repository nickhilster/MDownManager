// 01 · the problem

Markdown became the AI agent's
native file format. Nobody noticed.

Agent configuration, RAG corpora, system prompts, runbooks — all authored and consumed as .md. Yet no purpose-built home exists. And no one is screening what the agents read.

01

Fragmented storage

Markdown lives scattered across repos, Notion, cloud drives, and local folders. None of these index or serve .md specifically for agent retrieval.

02

Zero security layer

A .md file pulled from a repo or shared by a teammate can carry hidden instructions that hijack a downstream agent. Today this is screened by nobody — and agents now have tool access.

03

Wrong-shaped incumbents

Obsidian is editor-first. Notion locks content in a proprietary store. OneDrive syncs bytes, not Markdown structure. None are built for agent consumption or injection screening.

// 03 · product

Five capabilities. One coherent platform.

Each feature is independently valuable; together they form a complete security and management pipeline for the Markdown knowledge layer that agents consume.

F01

Core Vault

A local-first, personally hosted .md file server. Files stay on-disk as plaintext. SQLite index + LanceDB vectors serve them to agents in under 2 seconds. Optional remote sync via PocketBase — no third-party cloud required.

Core Infrastructure
Acceptance Criteria Store, retrieve & serve an .md file to an AI agent in <2s from local or remote vault.
F02

Prompt Injection Scanner

Three-layer detection: static pattern matching → LLM semantic analysis → context reasoning. Flags adversarial instructions, role confusion, tool injection, exfiltration attempts, and obfuscation. Line-level highlights with single-click navigation. Fully explainable verdicts.

Priority · Security
Acceptance Criteria ≥90% precision on defined test corpus. Each flag navigable in one UI interaction.
F03

AI Summary Generator

Structured executive summaries on demand: TL;DR, key bullets, decisions. Routes to local LLM by default; cloud LLM opt-in per feature with explicit disclosure. Sub-10-second latency target.

AI · Configurable Routing
Acceptance Criteria Summary in <10s. ≥80% of beta testers rate it useful.
F04

Structure Explorer

Tabulated view of every structural element — headings, code blocks, links, callouts, tasks. Sortable, filterable, exportable. Virtualised rendering handles 10,000+ line documents. Click-to-jump to source line.

UX · Performance
Acceptance Criteria Full structure interactive in <1.5s for files up to 500KB.
F05

Categorisation Engine

AI-first grouping via embeddings. Full manual drag-and-drop override that persists. One-click re-sort that respects or optionally resets manual changes. Manual labels always win over AI in conflict resolution.

AI + Manual Override
Acceptance Criteria ≥75% user agreement on initial AI grouping. Re-sort in <5s.
// 04 · technical strategy

Stack chosen for sovereignty
and consumer hardware

Every decision is optimised for one constraint: the app must coexist with a local LLM on the same machine, with zero mandatory cloud dependency.

Decision Options Evaluated Recommendation + Rationale
Desktop framework Electron, Flutter Desktop Tauri 2.x 12MB bundle vs 180MB Electron. ~85MB idle RAM vs ~450MB. 93% size reduction confirmed in 2026 benchmarks. Electron's Chromium runtime directly competes with the local LLM for memory.
LLM abstraction LangChain, custom LlamaIndex + thin custom router LlamaIndex owns indexing/RAG. Custom router handles per-feature local/cloud routing — keeps the security-critical injection path transparent and auditable. LangChain rejected: too abstracted to audit.
Local LLM runtime LM Studio, llama.cpp Ollama (default) + llama.cpp (embeddable) Ollama: simplest install, clean HTTP API, broad model library. llama.cpp: bundled for a true offline mode requiring zero external install.
UI component library Radix, Ant Design shadcn/ui on Radix Copy-in components, no version lock, fully accessible primitives. TanStack Table for Structure Explorer virtualisation. Ant Design rejected: too opinionated visually.
File storage / sync PocketBase, Supabase local SQLite + filesystem (v1) → PocketBase (team) Files-on-disk stay authoritative. SQLite index is disposable and rebuildable. PocketBase (single Go binary) is the clean self-hosted path for remote/team vaults.
Vector search ChromaDB, Weaviate LanceDB Embedded, serverless, on-disk, Rust-native. Native LlamaIndex integration. Sub-20ms query times on 1M vectors. No separate server process. $30M Series A (June 2025) signals production maturity.
// 05 · defensibility

Why this is hard to replicate

[S]

Security as the wedge

The prompt injection scanner is the category-defining feature. Building a high-precision, three-layer, explainable detection engine takes real investment. First-mover advantage accumulates in the test corpus and the pattern library.

[P]

Privacy as a sales motion

Local-first with no mandatory cloud is a genuine differentiator in regulated industries and privacy-conscious teams. Network effects don't apply — but trust does, and trust compounds with a clean track record.

[A]

Agent-native architecture

The local HTTP/IPC serving layer and the LanceDB vector index are purpose-built for agent consumption. Generic file managers retrofit this; MdownManager ships it as a first principle.

[O]

Open-core flywheel

An open-source core with a documented, auditable security model builds community trust that is especially potent for a security product. The pattern library grows with contributions; the premium tier monetises hosted sync and cloud routing.
