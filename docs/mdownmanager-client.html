🔒 Security · Priority Feature

Prompt Injection Scanner

Hidden instructions in a .md file can hijack an AI agent — causing it to leak data, execute commands, or ignore its own rules. MdownManager scans every file with three detection layers: fast pattern matching, LLM semantic analysis, and context reasoning. Every flag comes with the exact line number, a one-click jump, and a plain-language explanation of what was found and why it's risky.

📁 Core Vault

Your local-first .md file server

Like OneDrive — but purpose-built for Markdown and AI. Add folders, and MdownManager auto-discovers, indexes, and versions every .md file. A local vector database makes semantic search instant. An optional self-hosted remote vault enables sync across devices, with no third-party cloud involved.

<2s to retrieve and serve any .md file to an agent
✦ AI Summary Generator

TL;DR any file in seconds

Get a structured summary — TL;DR, key bullets, decisions — for any .md on demand. Runs against a local model by default; switch to a cloud provider for longer or complex files. When cloud is used, you're told exactly what's being sent and to whom, before it goes.

<10s summary latency target on local LLM
≡ Structure Explorer

See every element, sorted and filtered

A tabulated view of every heading, code block, link, callout, and task in a file. Sort by type or line number. Filter to just code blocks or just external links. Click any row to jump to that line. Works on 10,000-line documents without lag.

<1.5s full structure rendered for files up to 500KB
⊞ Categorisation Engine

AI-organised, human-controlled

AI groups your files on first run based on semantic similarity. Disagree? Drag-and-drop to reorganise. Your changes persist and survive any re-sort — AI suggestions only ever apply to files you haven't manually placed. One click to let the AI try again, with your changes always respected by default.

≥75% user agreement with AI's initial groupings in testing

[D]

Files stay on your disk

Your .md files remain as plaintext on your filesystem. MdownManager builds a disposable, rebuildable index — the files themselves are always the source of truth.

[N]

No telemetry by default

Crash reports and usage analytics are opt-in, not opt-out. Scan results and audit logs stay local unless you explicitly export them.

[O]

Fully offline capable

With Ollama or the embedded llama.cpp runtime, MdownManager works with zero internet connection. No mandatory cloud account, no sign-up required to use core features.

[E]

Explicit egress disclosure

When any feature routes to a cloud LLM, you see exactly what content is being sent, to which provider, before it goes. No silent uploads.
