Features

Everything your vault needs

Built for developers and power users who live in Markdown — from local notes to GitHub-hosted skill libraries.

Vault indexing

Recursively indexes .md files with FTS5 full-text search, content hash tracking, and live re-index on demand.

Semantic search

Generate vector embeddings with Ollama or OpenAI, then search by meaning — not just keywords.

AI summaries

Batch-summarize your vault with any model: Ollama, Anthropic, OpenAI, DeepSeek, or Google.

Security scanner

14 built-in rules detect secrets, credentials, and dangerous SKILL.md tool requests. Gitleaks-compatible.

GitHub import

Clone any public repo directly as a vault — no git binary required. Great for skill libraries like wshobson/agents.

Local HTTP API

Bearer-auth Axum server on port 7734 exposes your vault data to scripts, agents, and other tools.

Agent skill support

First-class SKILL.md detection

MDownManager understands the Claude Agent SDK skill format. Files named SKILL.md are automatically detected, validated for required frontmatter fields, and badged in the file table. The built-in scanner flags any skill that requests high-privilege tools like bash, write_file, or eval.

✓ Skill — name + description present
Skill — missing required fields
High risk — dangerous allowed-tools

HTTP API

Scriptable from anywhere

The embedded Axum server lets any local process — agents, scripts, CI pipelines — query your vault over HTTP.

GET /files?vault_id={id}&category=skill
GET /search?vault_id={id}&q={query}
GET /health

All endpoints require Authorization: Bearer <key>. The key is auto-generated and shown in Settings.
